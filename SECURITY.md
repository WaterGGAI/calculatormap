# Security Policy

## Supported Versions

Security fixes are currently applied against the latest `main` branch state in this repository.

| Version | Supported |
| --- | --- |
| `main` | yes |
| older commits and forks | no |

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

When reporting a potential vulnerability, include:

- a short description of the issue
- affected routes, files, or workflows
- clear reproduction steps
- impact details if known
- logs, screenshots, or proof-of-concept material when safe to share privately

If GitHub private vulnerability reporting is enabled for this repository, use that path first. Otherwise, contact the maintainer privately through GitHub rather than posting details in public.

## What to Expect

We will try to:

- acknowledge the report
- validate the issue
- work on a fix or mitigation
- publish the fix in the repository once it is safe to do so

Response times may vary, but private reporting is the fastest and safest path.
