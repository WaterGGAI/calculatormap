# CalculatorMap Roadmap

This roadmap reflects the current public direction for the project. It is intentionally practical and focused on shipping a useful calculator platform rather than a broad wishlist.

## Near term

- expand calculator coverage in finance, work, home, health, and business
- improve calculator-to-guide internal linking
- polish `zh-TW` localization so translated pages read naturally end to end
- improve desktop and mobile visual consistency across calculator templates
- add better validation and helper copy for complex calculator inputs
- improve admin editing workflows for content overrides

## Content and SEO

- strengthen cluster pages as decision hubs
- expand article coverage tied to calculator intent
- improve structured data output for calculators and guides
- make AI SEO automation easier to audit and review
- add stronger freshness and regeneration controls

## Platform

- expand automated testing around critical routes and content generation
- reduce framework warnings and keep the project aligned with newer Next.js conventions
- improve Cloudflare onboarding for contributors using their own accounts
- make local development and preview flows more predictable
- add more contributor-facing docs around D1 data and content overrides

## Admin and automation

- improve run history visibility and failure diagnostics
- add stronger review states for generated content
- make media and internal link workflows more complete
- improve category and cluster management tools
- add better safeguards around automated publishing decisions

## Open source maturity

- add more issue templates and contributor examples
- grow the docs set for architecture and data flow
- add a changelog or release notes process
- document extension points for adding new calculators and content types

## Out of scope for the public repo

- live production secrets
- private Cloudflare resource identifiers
- any internal-only operational credentials or unpublished business data
